למה זה משנה? מה אתם מפסידים?

🔍

כאשר משתמשים שואלים ChatGPT, Perplexity או Google AI Overviews על שירותי סיעוד - ManpowerCare לא יופיע בתשובות.

🛠

הגבלת כלי SEO

כלי ניתוח SEO מבוססי AI לא יכולים לנתח את התוכן באתר, מה שמקשה על אופטימיזציה.

🕑

בעיה שרק תחמיר

ככל שחיפוש AI יהפוך לנפוץ יותר, הנזק מחסימה רק ילך ויגדל. חשוב לפתור עכשיו.

מה גורם לחסימה?

הסיבות האפשריות לשגיאת 403 Forbidden

1

Cloudflare Bot Fight Mode

מצב הגנה אגרסיבי של Cloudflare שחוסם אוטומטית כל תנועה שנראית כמו בוט, כולל בוטים לגיטימיים של AI.

סבירות גבוהה
2

Security Level גבוה מדי

הגדרת Security Level ב-Cloudflare על "I'm Under Attack" או "High" חוסמת גישה לבוטים.

סבירות גבוהה
3

תוסף אבטחה ב-WordPress

תוספי אבטחה כמו Wordfence, Sucuri, All In One Security עשויים לחסום User-Agents של AI.

סבירות בינונית
4

חוקי WAF (Web Application Firewall)

חוקי Firewall ברמת השרת או Cloudflare שחוסמים User-Agents מסוימים או תבניות בקשה.

סבירות בינונית
5

הגדרות .htaccess

חוקים בקובץ .htaccess שחוסמים גישה על בסיס User-Agent או כתובות IP.

סבירות בינונית

פתרונות - איך לפתוח את האתר ל-AI

בחרו את הפתרון המתאים לפי הסיבה לחסימה

☁ הגדרות Cloudflare

1

כבו את Bot Fight Mode

Security → Bots → Bot Fight Mode → OFF

2

הורידו את Security Level

Security → Settings → Security Level → Medium (או Low)

3

צרו Firewall Rules לאפשר בוטי AI

Security → WAF → Custom Rules → Create Rule

# Rule Name: Allow AI Bots # Expression: (http.user_agent contains "GPTBot") or (http.user_agent contains "ClaudeBot") or (http.user_agent contains "Claude-Web") or (http.user_agent contains "Google-Extended") or (http.user_agent contains "PerplexityBot") or (http.user_agent contains "Amazonbot") # Action: Skip (all remaining rules)

🔒 תוספי אבטחה ב-WordPress

1

Wordfence

Wordfence → Firewall → Blocking → Whitelist AI User-Agents

# Add to Whitelist patterns: GPTBot ClaudeBot Claude-Web Google-Extended PerplexityBot
2

Sucuri

Sucuri → Firewall (WAF) → Whitelist → Add User-Agents

3

All In One WP Security

WP Security → Firewall → Blacklist → וודאו שבוטי AI לא חסומים

📝 קובץ robots.txt

1

וודאו שאין חסימה של בוטי AI

בדקו את הקובץ ב: manpowercare.co.il/robots.txt

# robots.txt מומלץ - מאפשר גישה לבוטי AI User-agent: GPTBot Allow: / User-agent: ClaudeBot Allow: / User-agent: Claude-Web Allow: / User-agent: Google-Extended Allow: / User-agent: PerplexityBot Allow: / User-agent: * Allow: /

📄 קובץ .htaccess

1

בדקו אם יש חוקי חסימה

חפשו שורות שחוסמות User-Agents והסירו או עדכנו אותן

# אם יש חוקי חסימה כאלה - הסירו אותם: RewriteCond %{HTTP_USER_AGENT} (bot|crawl|spider) [NC] RewriteRule .* - [F,L] # או הוסיפו חריגים לבוטי AI: RewriteCond %{HTTP_USER_AGENT} !GPTBot [NC] RewriteCond %{HTTP_USER_AGENT} !ClaudeBot [NC] RewriteCond %{HTTP_USER_AGENT} !Google-Extended [NC]
