הערכת סיכוני דירוג

סיכון קריטי: מבנה H1 בדף הבית

17 תגיות H1 בדף הבית מבלבלות את Google לגבי נושא העמוד הראשי. זה עלול לפגוע בדירוג עבור מילות מפתח כמו "מכללת בנייה", "קורסים בנייה", "לימודי בנייה".

סיכון גבוה: עמוד המכללה ללא H1 ו-Meta

עמוד /building-college/ הוא עמוד מפתח לדירוג, אך חסרים בו H1 ו-Meta Description. Google לא מקבל אותות ברורים לגבי תוכן העמוד.

סיכון בינוני: Meta Description חסר ב-75% מהעמודים

Google יוצר snippet אוטומטי שלא בהכרח אופטימלי. זה פוגע ב-CTR בתוצאות החיפוש.

רשימת תיקונים מעודכנת - לפי עדיפות

  1. 1

    תקן את מבנה H1 בדף הבית - מיידי

    שנה את כל הכותרות מ-H1 ל-H2/H3 מלבד כותרת ראשית אחת. כרגע יש 17 H1 - צריך להיות רק אחד כמו "מכללת מרכז הבנייה הישראלי".

  2. 2

    הוסף H1 ו-Meta Description לעמוד המכללה

    עמוד /building-college/ חייב H1 כמו "מכללת מרכז הבנייה הישראלי" ותיאור מטא ייחודי.

  3. 3

    תקן 404 של Cloudflare Email Protection

    138 קישורים פנימיים מצביעים ל-/cdn-cgi/l/email-protection שמחזיר 404. בדוק הגדרות Cloudflare או הסר קישורים.

  4. 4

    תקן מבנה H1 בעמוד כנסים

    עמוד /conferences/ מכיל 13 H1 - כל כותרת אירוע מוגדרת כ-H1. שנה ל-H2/H3.

  5. 5

    הוסף Meta Description לכל העמודים המרכזיים

    עמודים חסרים: /building-college/, /contact/, /conferences/, /events/, /magazine/, /blog/

  6. 6

    הוסף H1 לעמוד צור קשר

    עמוד /contact/ חסר H1 לחלוטין. הוסף כותרת כמו "צור קשר - מרכז הבנייה הישראלי".

  7. 7

    עדכן קישור 301 בפוטר

    הקישור /articles/קורס-קבלני-אינסטלציה/ מפנה ל-URL ארוך יותר. עדכן ישירות ל-URL הסופי.

מה תוקן בהצלחה

סכמות מובנות (Structured Data) - הותקנו!

כל העמודים כוללים כעת:

  • Organization Schema - מידע על הארגון
  • WebSite Schema - עם SearchAction לחיפוש
  • BreadcrumbList Schema - ניווט פירורי לחם
  • WebPage Schema - מידע על כל עמוד

זה שיפור משמעותי שיאפשר הצגת Rich Snippets בתוצאות החיפוש.

  • 2. הוסף Course Schema - עמודי הקורסים צריכים סכמת Course לשיפור הנראות בחיפוש
