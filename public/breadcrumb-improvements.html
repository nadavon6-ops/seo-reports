Issue Found

The /showrooms/ page has correct BreadcrumbList schema, but the /models/ page and some other sections are missing breadcrumb schema entirely. This creates inconsistent implementation across the site.

Solution

Enable breadcrumbs globally in Yoast SEO or RankMath for ALL page types and custom post types. The "models" post type likely has breadcrumbs disabled in the SEO plugin settings.

WordPress Fix Steps

  • Yoast SEO: Go to SEO → Search Appearance → Content Types → Models → Enable breadcrumbs
  • RankMath: Go to Rank Math → General Settings → Breadcrumbs → Enable for all post types
  • Check that the theme template includes the breadcrumb shortcode: [wpseo_breadcrumb] or rank_math_the_breadcrumbs()
  • Verify fix using Google Rich Results Test on 2-3 model pages

How to Verify Breadcrumbs

  • Step 1: Visit an internal page (not homepage)
  • Step 2: Right-click → View Page Source → Search for "BreadcrumbList"
  • Step 3: Or use Google Rich Results Test: https://search.google.com/test/rich-results
  • Step 4: Verify the breadcrumb path matches the page hierarchy (e.g., Home → Category → Page)
  • Step 5: Confirm the last item has NO "item" property (only "name")
