מידע בלבד סעיף 4: פערי תוכן מול מתחרים

הבהרה: סעיף זה הוא אינפורמטיבי בלבד ואינו מצריך פעולה מצוות הפיתוח בשלב זה. המידע מוצג להבנת ההקשר העסקי.

סיכום פערי התוכן

בהשוואה לקופות החולים המתחרות (כללית, מכבי, מאוחדת), לאומית חסרים מספר נושאי תוכן משמעותיים:

נושא מצב בלאומית מצב אצל מתחרים הערות
סוכרת (מדריך מקיף) עמוד סדנה אחד 2-7 עמודים לכל קופה נושא עם ביקוש גבוה (40,500+ חיפושים/חודש)
בריאות הנפש אזכור בודד 8+ עמודים ייעודיים ביקוש גבוה מאוד (88,000+ חיפושים/חודש)
חיסונים מינימלי 7+ מדריכים + לוח חיסונים הזדמנות לתפוס שאילתות רפואיות
מחשבון הריון לא קיים קיים במכבי ובמאוחדת נפח חיפוש גבוה
מחשבון BMI לילדים לא קיים קיים במכבי 2,400 חיפושים/חודש
"עולמות" תוכן (כמו במכבי) לא קיים 8 האבים במכבי ("עולמות במכבי") אסטרטגיית תוכן מוכחת
לתשומת לב: סעיף זה אינו מצריך פעולה מצד צוות הפיתוח. יצירת תוכן חדש היא משימה של צוות התוכן/שיווק ותתואם בנפרד.

עדיפות בינונית סעיף 5: עמודים עם פוטנציאל אופטימיזציה

העמודים הבאים מציגים חשיפה גבוהה (impressions) אך שיעור הקלקה (CTR) נמוך. שיפור ה-Meta Tags יכול להגדיל משמעותית את התנועה ללא שינוי בדירוג.

סיכום משימות לצוות הפיתוח

  • 7 עמודים דורשים עדכון Meta Title ו-Meta Description
  • 4 עמודים דורשים הוספת בלוקי FAQ
  • צפי השפעה: +15-30% שיפור ב-CTR = +5,000 קליקים/חודש

עדיפות גבוהה - קריטי סעיף 6: הטמעת Schema Markup

חשיבות קריטית: הטמעת Schema היא המשימה החשובה ביותר בסעיף זה. Schema מאפשר הצגת Rich Results בגוגל, הגדלת CTR, ותפיסת Featured Snippets.

סיכום הטמעות Schema נדרשות

סוג Schema עמודים עדיפות
BreadcrumbList כל העמודים באתר קריטי
Organization עמוד הבית + אודות גבוהה
FAQPage כל עמודי התוכן הרפואי (50+) קריטי
MedicalWebPage מדריכים רפואיים בינונית

קריטי 1. BreadcrumbList Schema - הטמעה כלל-אתרית

מה זה: סימון מבנה הניווט (פירורי לחם) לגוגל

למה חשוב: משפר הצגה בתוצאות החיפוש, מקל על משתמשים

היכן ליישם: כל העמודים באתר

כללי יישום:
  • כל עמוד צריך BreadcrumbList ייחודי משלו
  • הפריט האחרון (העמוד הנוכחי) - ללא שדה "item" (רק name)
  • position חייב להיות רציף: 1, 2, 3...
  • כל ה-URLs חייבים להיות מלאים (כולל https://)
  • שמות בעברית
תבנית קוד:
<script type="application/ld+json"> { "@context": "https://schema.org", "@type": "BreadcrumbList", "itemListElement": [ { "@type": "ListItem", "position": 1, "name": "דף הבית", "item": "https://www.leumit.co.il/" }, { "@type": "ListItem", "position": 2, "name": "מחלות", "item": "https://www.leumit.co.il/diseases/" }, { "@type": "ListItem", "position": 3, "name": "חצבת" } ] } </script>
שימו לב: הפריט האחרון (העמוד הנוכחי) מכיל רק "name" ללא "item"!

בינונית 4. MedicalWebPage Schema - מדריכים רפואיים

מה זה: סימון עמוד כתוכן רפואי

למה חשוב: עוזר לגוגל לסווג נכון תוכן רפואי

היכן ליישם: מדריכי מחלות, מידע רפואי

תבנית קוד:
<script type="application/ld+json"> { "@context": "https://schema.org", "@type": "MedicalWebPage", "name": "חצבת - מידע, תסמינים וחיסון", "description": "מידע מקיף על מחלת החצבת - תסמינים, דרכי הדבקה, חיסון ומניעה", "url": "https://www.leumit.co.il/diseases/measles/", "inLanguage": "he-IL", "datePublished": "2024-01-01", "dateModified": "2026-01-20", "publisher": { "@type": "Organization", "name": "לאומית שירותי בריאות", "@id": "https://www.leumit.co.il/#organization" }, "medicalAudience": { "@type": "MedicalAudience", "audienceType": "Patient" }, "specialty": { "@type": "MedicalSpecialty", "name": "Infectious Disease" } } </script>

הנחיות יישום כלליות

מיקום הקוד:
  • כל ה-Schema צריך להיות בתוך תג <script type="application/ld+json">
  • מיקום מומלץ: לפני סגירת </head> או לפני סגירת </body>
  • ניתן לשלב מספר Schemas באותו עמוד (כל אחד בתג script נפרד)
בדיקה ואימות:
  • לפני העלאה - לבדוק ב: schema.org/validator
