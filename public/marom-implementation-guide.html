הטמעת מפת גוגל

ללקוח יש משרדים פיזיים. הוספת מפת גוגל אינטראקטיבית לדף הבית משפרת את סיגנלי ה-Local SEO ומסייעת ללקוחות פוטנציאליים למצוא את המשרד בקלות.

מיקומי המשרדים

משרד תל אביב (ראשי)

כתובת: הארד 9, רמת החייל, תל אביב

משרד ירחיב

כתובת: אנפה 17, מושב ירחיב

רשימת משימות ליישום

  • יצירת קוד embed של מפת גוגל עבור משרד תל אביב מ-Google Maps
  • יצירת קומפוננטת מפה בדף הבית (מתחת ל-hero או מעל הפוטר)
  • הוספת קישור ניווט ל-Waze לצד המפה
  • וידוא טעינה עצלה של המפה לביצועים (loading="lazy")
  • בדיקת רספונסיביות במובייל
  • עדכון LocalBusiness schema עם קואורדינטות geo

תבנית HTML להטמעה

<!-- מדור מפת גוגל --> <section class="map-section"> <h2>מיקום המשרד</h2> <div class="map-container"> <iframe src="https://www.google.com/maps/embed?pb=!1m18!1m12!..." width="100%" height="400" style="border:0; border-radius: 12px;" allowfullscreen="" loading="lazy" referrerpolicy="no-referrer-when-downgrade" title="משרד מרום גולי שמאות מקרקעין"> </iframe> </div> <p> <strong>כתובת:</strong> הארד 9, רמת החייל, תל אביב <a href="https://waze.com/ul?q=הארד 9 רמת החייל תל אביב" target="_blank"> נווט עם Waze </a> </p> </section>

עדכון Schema נדרש

יש להוסיף ל-LocalBusiness schema הקיים:

"geo": { "@type": "GeoCoordinates", "latitude": "32.1234", "longitude": "34.8234" }, "hasMap": "https://maps.google.com/?q=הארד+9+רמת+החייל+תל+אביב"

עדיפות גבוהה
מדור ביקורות לקוחות

ללקוח יש פרופיל Google My Business פעיל. הוספת מדור ביקורות לקוחות עם Review schema מאפשרת rich snippets בתוצאות החיפוש ומגבירה את אמון המשתמשים.

רשימת משימות ליישום

  • קבלת כתובת URL של פרופיל Google My Business מהלקוח
  • איסוף 3-5 ביקורות מאושרות להצגה
  • יצירת קומפוננטת ביקורות עם תמיכת RTL
  • עיצוב כרטיסי ביקורת עם דירוג כוכבים (צבע זהב #FFD700)
  • הוספת תג דירוג גוגל המציג ציון כולל
  • יישום כפתור "כתבו ביקורת" המקשר ל-GMB
  • הוספת AggregateRating schema
  • הוספת Review schemas בודדים (מינימום 3)
  • אימות schema ב-Google Rich Results Test

תבנית HTML לכרטיס ביקורת

<section class="reviews-section" dir="rtl"> <h2>מה לקוחותינו אומרים</h2> <!-- תג דירוג גוגל --> <div class="google-rating"> <img src="/google-logo.png" alt="Google"> <span class="rating">4.9</span> <span class="stars">★★★★★</span> <span>(XX ביקורות)</span> </div> <!-- גריד כרטיסי ביקורות --> <div class="reviews-grid"> <div class="review-card"> <div class="review-header"> <div class="avatar">א</div> <div> <strong>שם הלקוח</strong> <span class="stars">★★★★★</span> </div> </div> <p>"תוכן הביקורת כאן..."</p> <span class="date">לפני 3 חודשים</span> </div> </div> <!-- כפתור CTA --> <a href="GMB_REVIEW_LINK" class="review-cta"> כתבו לנו ביקורת בגוגל → </a> </section>

Schema נדרש

{ "@context": "https://schema.org", "@type": "LocalBusiness", "name": "מרום גולי שמאות מקרקעין בע״מ", "aggregateRating": { "@type": "AggregateRating", "ratingValue": "4.9", "reviewCount": "XX", "bestRating": "5", "worstRating": "1" }, "review": [ { "@type": "Review", "author": { "@type": "Person", "name": "לקוח מאומת" }, "reviewRating": { "@type": "Rating", "ratingValue": "5" }, "reviewBody": "טקסט הביקורת כאן...", "datePublished": "2025-XX-XX" } ] }

הערה חשובה

הביקורות המוצגות חייבות להיות אותנטיות מפרופיל ה-GMB האמיתי של הלקוח. יש לקבל אישור בכתב מהלקוח לפני פרסום ביקורות לקוחות באתר.

עדיפות גבוהה
קישורי רשתות חברתיות

לאתר אין כרגע קישורים לרשתות חברתיות - זהו אלמנט חסר משמעותי. סיגנלים חברתיים תורמים לסמכות המותג ומספקים ערוצים נוספים למעורבות לקוחות.

פלטפורמות נדרשות

פלטפורמה עדיפות מטרה פעולה נדרשת
LinkedIn קריטי רשת מקצועית, לידים B2B לקבל URL מהלקוח
Facebook גבוהה קהל מקומי, סיפורי הצלחה לקבל URL מהלקוח

רשימת משימות ליישום

  • אישור כתובות URL של פרופילי הרשתות החברתיות מול הלקוח
  • הוספת אייקונים חברתיים לכותרת (קטנים, מיושרים לשמאל)
  • יצירת מדור רשתות חברתיות בפוטר
  • שימוש באייקוני SVG לאיכות מיטבית
  • הוספת hover states עם צבעי המותג של הפלטפורמות
  • וידוא פתיחת קישורים בטאב חדש (target="_blank")
  • הוספת rel="noopener noreferrer" לאבטחה
  • הוספת aria-label לנגישות
  • עדכון Organization schema עם sameAs URLs

<!-- קישורי רשתות חברתיות - כותרת --> <div class="social-links-header"> <a href="LINKEDIN_URL" target="_blank" rel="noopener noreferrer" aria-label="LinkedIn"> <!-- אייקון LinkedIn SVG --> <svg width="20" height="20" viewBox="0 0 24 24" fill="currentColor"> <path d="M20.447 20.452h-3.554v-5.569c0-1.328..."></path> </svg> </a> <a href="FACEBOOK_URL" target="_blank" rel="noopener noreferrer" aria-label="Facebook"> <!-- אייקון Facebook SVG --> </a> </div>

יישום בפוטר

<!-- קישורי רשתות חברתיות - פוטר --> <div class="footer-social"> <p>עקבו אחרינו:</p> <div class="social-icons"> <a href="LINKEDIN_URL" target="_blank" rel="noopener"> LinkedIn </a> <a href="FACEBOOK_URL" target="_blank" rel="noopener"> Facebook </a> </div> </div>

עדכון Schema נדרש

יש להוסיף sameAs ל-Organization schema הקיים:

{ "@type": "Organization", "name": "מרום גולי שמאות מקרקעין בע״מ", "sameAs": [ "https://www.linkedin.com/in/marom-guli-XXXXX/", "https://www.facebook.com/maromguli.appraiser/" ] }

פעולה נדרשת מהלקוח

לפני היישום, יש לאשר את הדברים הבאים מול הלקוח:

  • האם יש לו פרופיל LinkedIn? אם לא, מומלץ ליצור (חיוני ל-B2B)
  • האם יש לו עמוד עסקי בפייסבוק?
  • לקבל את כתובות ה-URL המדויקות לכל הפרופילים הפעילים
