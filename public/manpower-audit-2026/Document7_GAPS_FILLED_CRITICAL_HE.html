1. JobPosting Schema קריטי ביותר

למה זה הפער הקריטי ביותר?

מנפאואר היא חברת השמה עם אלפי משרות פתוחות. JobPosting Schema הוא הסכמה המזהה של Google לחברות תעסוקה. ללא סכמה זו:

  • המשרות לא יופיעו ב-Google Jobs (תוצאות משרות מורחבות)
  • מתחרים עם JobPosting יקבלו יותר נראות
  • CTR נמוך יותר ב-30-50% לדפי משרות

1.3 הטמעה דינמית (Next.js)

מאחר שמנפאואר משתמשת ב-Next.js, הנה דוגמה להטמעה דינמית:

// pages/jobs/[jobId].tsx
import Head from 'next/head';

export default function JobPage({ job }) {
  const jobPostingSchema = {
    "@context": "https://schema.org",
    "@type": "JobPosting",
    "title": job.title,
    "description": job.description,
    "datePosted": job.postedDate,
    "validThrough": job.expiryDate,
    "employmentType": job.employmentType,
    "hiringOrganization": {
      "@type": "Organization",
      "name": "מנפאואר ישראל",
      "sameAs": "https://www.manpower.co.il",
      "logo": "https://manpowercontent.manpower.co.il/wp-content/uploads/2024/05/Manpower-logo-horizontal.svg"
    },
    "jobLocation": {
      "@type": "Place",
      "address": {
        "@type": "PostalAddress",
        "addressLocality": job.location,
        "addressCountry": "IL"
      }
    },
    "directApply": true
  };

  return (
    <>
      <Head>
        <script
          type="application/ld+json"
          dangerouslySetInnerHTML={{ __html: JSON.stringify(jobPostingSchema) }}
        />
      </Head>
      {/* Rest of job page content */}
    </>
  );
}
השפעה צפויה:
  • הופעה ב-Google Jobs (תוצאות משרות מורחבות)
  • עלייה של 40-60% בקליקים על משרות
  • Rich Snippets עם פרטי משרה בתוצאות חיפוש
  • יתרון תחרותי מול סוכנויות ללא Schema

3. robots.txt - ניתוח והמלצות

3.1 המלצה ל-robots.txt

מיקום: https://www.manpower.co.il/robots.txt

# robots.txt for www.manpower.co.il
# Last updated: January 2026

User-agent: *
Allow: /

# Block admin and API routes
Disallow: /api/
Disallow: /_next/
Disallow: /admin/
Disallow: /login/
Disallow: /dashboard/

# Block search results pages (prevent thin content indexing)

Disallow: /*?filter=

# Allow important crawlers full access
User-agent: Googlebot
Allow: /

Allow: /

# Sitemap location

Sitemap: https://www.manpower.co.il/sitemap-articles.xml

3.2 בדיקות נדרשות

בדיקה מה לוודא כלי
robots.txt נגיש מחזיר 200 ולא 404 Browser / curl
Sitemap מקושר Sitemap directive קיים robots.txt
עמודים חשובים לא חסומים דפי שירות, מאמרים, משרות נגישים GSC URL Inspection
עמודים פנימיים חסומים Admin, API, Login חסומים robots.txt tester
אזהרה: אם robots.txt לא קיים או שגוי, Google עלול לאנדקס עמודים פנימיים (admin, API) שעלולים להוות סיכון אבטחה וליצור תוכן כפול.

4. Core Web Vitals - המלצות ספציפיות

4.1 יעדי ביצועים

מדד יעד טוב צריך שיפור גרוע
LCP (Largest Contentful Paint) < 2.5 שניות 2.5-4 שניות > 4 שניות
INP (Interaction to Next Paint) < 200ms 200-500ms > 500ms
CLS (Cumulative Layout Shift) < 0.1 0.1-0.25 > 0.25

4.2 המלצות לשיפור LCP

  1. תמונות: השתמשו ב-WebP/AVIF במקום JPG/PNG
  2. Lazy Loading: רק לתמונות מתחת ל-fold. תמונת Hero צריכה להיטען מיד
  3. Preload: הוסיפו preload לתמונת ה-LCP: 
    <link rel="preload" as="image" href="/hero-image.webp">
  4. CDN: ודאו שכל התמונות נטענות מ-CDN (כבר יש לכם manpowercontent.manpower.co.il)
  5. Server Response: יעד < 200ms. אם איטי, בדקו hosting ו-caching

4.3 המלצות לשיפור INP

  1. JavaScript: הקטינו bundle size, השתמשו ב-code splitting
  2. Third-party scripts: טענו Google Analytics/Tag Manager באופן async
  3. Event handlers: הימנעו מ-long tasks (>50ms)

4.4 המלצות לשיפור CLS

  1. תמונות: הגדירו תמיד width ו-height: 
    <img src="image.jpg" width="800" height="450" alt="תיאור">
  2. פונטים: השתמשו ב-font-display: swap ו-preload: 
    <link rel="preload" href="/fonts/hebrew.woff2" as="font" type="font/woff2" crossorigin>
  3. מודעות/Embeds: שמרו מקום מראש עם min-height

4.5 בדיקה וניטור

  • PageSpeed Insights: https://pagespeed.web.dev/?url=https://www.manpower.co.il
  • GSC Core Web Vitals Report: נתונים אמיתיים מ-Chrome UX Report
  • Lighthouse: בדיקה מקומית ב-Chrome DevTools

5. אסטרטגיית Backlinks

5.1 מצב נוכחי (הערכה)

מנפאואר ישראל הוא brand מוכר עם 65 שנה בשוק. סביר להניח שיש בסיס טוב של backlinks, אך יש לבצע בדיקה מלאה.

קטגוריה דוגמאות עדיפות

5.3 אסטרטגיות לבניית קישורים

  1. מחקר תעסוקה: פרסום מחקרים ודוחות שנתיים על שוק העבודה (נתונים מקוריים = linkable assets)
  2. מומחיות תקשורתית: הופעת נציגי מנפאואר כמומחי תעסוקה בתקשורת
  3. שיתופי פעולה: עם אוניברסיטאות, מכללות ותוכניות הכשרה
  4. אירועים: כנסי תעסוקה וירידי משרות (יוצרים coverage ו-links)
  5. Guest Posts: מאמרים מקצועיים בפורטלים עסקיים

5.4 מה לא לעשות

  • לא לקנות קישורים (מפר את הנחיות Google)
  • לא להשתתף ב-link schemes
  • לא לבנות קישורים מאתרי ספאם או PBNs
  • לא להשתמש ב-exact match anchor text באופן מוגזם

6. אסטרטגיית ויקיפדיה וציטוטים

  • משפר E-E-A-T signals
  • יוצר "ידע מקושר" שמערכות AI מזהות

6.2 בדיקת מצב נוכחי

לבדוק:

  1. האם קיים ערך "מנפאואר ישראל" בויקיפדיה העברית?
  2. האם מנפאואר מוזכרת בערכים קשורים (כוח אדם, השמה, תעסוקה)?
  3. האם יש קישורים חיצוניים לאתר מנפאואר מויקיפדיה?
